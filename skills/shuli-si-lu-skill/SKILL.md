# 梳理思路 Skill

这个 Skill 是原 Superpowers 思路的中文调用名。核心原则是：

> 流程大于提示词，先想清楚，再开始做。

## 使用时机

当用户的想法还比较散、需求还不够清楚、项目还没有拆成步骤时，使用这个 Skill。

典型场景：

- 梳理一个模糊想法。
- 拆解一个新项目。
- 设计知识库结构。
- 规划课程、产品、服务或训练营。
- 设计 AI Agent 或自动化工作流。
- 把一堆素材、需求、目标收束成可执行方案。

## 工作流程

1. 先复述当前理解

用 2-4 句话说清楚你理解到的目标、用户、背景和最终交付物。

2. 找出关键缺口

判断现在最缺的是目标、人群、场景、资源、约束、交付物，还是执行顺序。

3. 一次最多问 3 个关键问题

问题要短、具体、能推动决策。不要为了显得专业而问太多。

如果上下文已经足够，不要硬问问题，直接进入拆解。

4. 拆成可执行路径

输出：

- 当前目标
- 关键假设
- 执行步骤
- 最小下一步
- 风险或待确认点

5. 收束到下一步动作

最后要让用户知道“现在先做什么”，而不是只得到一堆分析。

## 输出风格

- 直接、清楚、有条理。
- 少讲概念，多帮用户做判断。
- 不要长篇大论。
- 用户要的是想清楚和能行动，不是理论课。

## 默认口径

当用户说“调用梳理思路skill”时，先按这个结构开始：

```text
我先帮你把这件事梳理成可执行结构：

1. 当前目标：
2. 我理解的背景：
3. 现在最需要确认的 2-3 个问题：
4. 初步拆解路径：
5. 最小下一步：
```
